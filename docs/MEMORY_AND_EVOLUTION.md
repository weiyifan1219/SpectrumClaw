# 记忆与进化机制

## 参考方向

参考 AerialClaw 的记忆和进化系统，将 SpectrumClaw 的任务经验沉淀为可浏览、可复盘、可改进的系统能力。

## 记忆层规划

| 层 | 内容 | 示例 |
| --- | --- | --- |
| Working Memory | 当前任务上下文 | 用户问题、当前 skill、参数 |
| Episodic Memory | 任务执行经历 | 某次频率规划问题的执行结果 |
| Skill Memory | 能力经验 | 哪类查询适合哪个检索参数 |
| Domain Memory | 领域知识 | ITU 标准、频段规则、术语关系 |

## Evolution 规划

| 记录项 | 用途 |
| --- | --- |
| skill 成功率 | 判断能力稳定性 |
| 失败原因 | 辅助修复 prompt、脚本或依赖 |
| 用户反馈 | 指导后续迭代 |
| 参数漂移 | 发现检索参数或模型配置变化 |
| 版本摘要 | 形成系统进化记录 |

## 前端展示

Memory & Evolution 页面先展示静态规划，后续接入：

- 记忆条目列表。
- skill 使用次数和结果统计。
- 反思摘要。
- 系统能力版本变化。
- 用户确认后的改进建议。
