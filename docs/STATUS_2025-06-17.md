## 一、已完成事项

### 1. PDF 去重
- **工具**: `scripts/dedup_pdfs.py`
- **方法**: 基于完整文件内容 MD5 hash
- **结果**: 5276 → 4658 文件（618 个重复移到 `data/knowledge_base/duplicates/`）
- **状态**: ✅ 完成

### 2. MinerU 升级至 3.3.0
- **旧版**: magic-pdf 1.3.9（GPU 0% 利用率，无 equation/footnote 分类）
- **新版**: mineru 3.3.0（Python 3.11 + torch 2.6.0+cu124）
- **安装方式**: 通过本地 SSH 隧道代理，服务器 pip install
- **模型**: PDF-Extract-Kit-1.0 (9.3G) + MinerU2.5-Pro-2605-1.2B (2.2G, VLM 备选)
- **状态**: ✅ 完成

### 3. 全量文档 Pipeline 解析
- **引擎**: MinerU 3.3.0 pipeline 模式，双 GPU 常驻 API 服务
- **输出**: `data/parsed/<stem>/auto/<stem>_content_list.json`
- **块类型**: text / table / equation / chart / list / page_footnote / header / footer / page_number / image / code
- **结果**: 4656/4658 文件成功，2 个超大文件超时
- **脚本**: `scripts/batch_mineru_parse_api.py`
- **状态**: ✅ 基本完成（2 个文件失败）

### 4. ChromaDB 入库
- **入库脚本**: `scripts/ingest_parsed.py`
- **过滤策略**: 跳过 header / footer / page_number / image（保留 table/caption 文字内容）
- **去重**: `chroma_store.py` `add_blocks()` 入口过滤 + `search()` over-fetch + 去重
- **结果**:

| 指标 | 数值 |
|------|------|
| 入库 vectors | 1,399,856 |
| 嵌入 blocks | 1,399,955 |
| 跳过 blocks | 441,618 |
| 总 blocks | 1,841,474 |
| 总字符数 | 249.4M |
| 耗时 | 747 分钟 |

- **状态**: ✅ 完成

### 5. 知识图谱重建
- **方法**: Regex 提取 FrequencyBand / Standard / Footnote → Document 关系
- **结果**: 9,239 entities / 16,259 relations
- **Graph 文件**: `data/graph/spectrum_graph.json`
- **状态**: ✅ 完成

### 6. RAG 检索流程优化
- **Query 重写**: DeepSeek LLM 将中文 query 重写为搜索优化英文 query（集成到 graph nodes）
- **冷启动修复**: `app.py` startup 事件预加载 embedding 模型
- **动态统计**: retriever.py / tools.py / chat.py 不再硬编码 PDF 数量
- **状态**: ✅ 完成

### 7. 前端适配
- KB 统计: PDF 数量 / 向量数 / 知识图谱实体 / 文本块数量 动态显示
- 文档管理: 支持 4656 个文档列表 + 搜索 + 分页
- **状态**: ✅ 完成

### 8. 评测框架
- 100 题评测跑通（spectrumclaw_rag 全部 OK）
- Judge LLM 评分框架就绪

### 🟡 中等

| 卡点 | 描述 | 建议 |
|------|------|------|
| 2 个超大文件解析失败 | R-REC-M.1457-15-202010-I.pdf (1580页) 和 T-G.994.pdf (916页) 超时 | 用 txt 模式单独处理，或跳过（4656 文件已足够） |
| 评测 llm_only/vector_rag 方法报500 | 这两个方法在评测脚本中调 `/api/eval/llm_only` 端点，该端点依赖 LLM 隧道 | LLM 隧道稳定后重跑 |
| vector_count 冷启动为 0 | 已通过 startup 预热解决，但偶有反复 | 加 health check 依赖 |
| doc_registry 格式不兼容 | list → dict 格式需手动转换（脚本已处理） | 统一 registry 格式 |
| 补入库无去重逻辑 | `ingest_parsed.py` 不带 `--clear` 时会重复插入 | 加增量模式（检查 block_id 是否存在） |

### 🟢 后续优化

| 事项 | 优先级 | 建议 |
|------|--------|------|
| VLM 图片/chart 描述补录入库 | 中 | 用 Qwen-VL API（8241 隧道）或本地 MinerU2.5-Pro-1.2B 做 hybrid 解析 |
| Cross-encoder Rerank | 低 | 当前规则 Rerank 对频谱领域够用，后续可选 BGE-Reranker |

| 知识图谱 LLM 增强 | 低 | 参考 RAG-Anything，用 LLM 自动提取实体/关系替代 regex |
